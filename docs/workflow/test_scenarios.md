# Test Scenarios: MHWs Equipment Manager

本ドキュメントは、最新の仕様（SSoT）に基づき、エンジニアが単体テストおよび手動・E2Eテストを実施するためのシナリオを定義したものです。

## 1. ユニットテスト・ロジック検証 (Logic Unit Tests)

### 1.1. EquipmentBox (所持武器管理)
- **TS-EB-001: 正常な武器登録**:
  - **入力**: 武器種, 属性, スキル, 生産ボーナス, 復元ボーナス(2枠以下)。
  - **期待値**: 正常に登録され、IDが発行されること。データがストレージに保存されること。
- **TS-EB-002: 重複バリデーション (NG)**:
  - **入力**: 同一種類+同一レベルの復元ボーナスを3枠含めて登録・更新。
  - **期待値**: バリデーションエラーが発生し、保存を拒否すること。
- **TS-EB-003: 重複バリデーション (OK)**:
  - **入力**: 同一種類だがレベルが異なるボーナスを計3枠含めて登録（例：攻撃Ⅱx2 + 攻撃Ⅲx1）。
  - **期待値**: 正常に保存されること。

### 1.2. SkillUpgrade (スキル抽選トラッカー)
- **TS-SU-001: 強化の適用と武器更新**:
  - **入力**: 抽選結果A（スキルA）を「適用」、対象武器は武器B。
  - **期待値**: 
    1. 武器Bの現在のスキルがスキルAに更新される。
    2. 抽選結果Aがリストから削除される。
    3. 同じテーブル（武器種×属性）を共有する他の抽選結果Cの残り回数も減算される。
- **TS-SU-002: 自動削除**:
  - **入力**: 残り回数が1回の抽選結果に対して「1回分進行」を実行。
  - **期待値**: 残り回数が0以下となり、リストから自動的に削除されること。

### 1.3. RestorationTracker (復元厳選)
- **TS-RT-001: 強化の適用と自動削除**:
  - **入力**: 目標Aを「適用（実施）」。
  - **期待値**: 
    1. 紐付いている武器の復元ボーナス5枠が、目標Aの内容で更新される。
    2. 目標Aがリストから削除される。
    3. その武器に紐付く他の全目標の残り回数が減算され、0以下になったものは削除される。

### 1.4. Master Data & Normalization
- **TS-MD-001: レベル表記の正規化**:
  - **入力**: "1", "1.0", "EX", "無印"。
  - **期待値**: それぞれ "Ⅰ", "Ⅰ", "EX", "無印" として表示・管理されること。

### 1.5. Talismans (鑑定護石管理)
- **TS-TL-001: マスタデータに基づくスキルバリデーション (OK)**:
  - **入力**: レア度8で、マスタデータ (`talisman_master.json`) のパターン「B-A-G」に合致するスキル（例：砲術Lv2 - 攻撃Lv1 - 無我の境地Lv3）を指定して登録。
  - **期待値**: バリデーションを通過し、正常に保存されること。
- **TS-TL-002: マスタデータに基づくスキルバリデーション (NG)**:
  - **入力**: レア度8で、存在しないパターンや、グループ規定外のレベル（例：砲術Lv3 で Aグループ扱いとする等）を指定して登録。
  - **期待値**: マスタのエラー検知によりバリデーション違反となり、保存を拒否すること。
- **TS-TL-003: スロットのカスケードバリデーション**:
  - **入力**: レア度7のとき、マスターに存在しないレベルのスロットを入力。
  - **期待値**: 保存をブロックすること（ただしUIの正常なカスケード制御により、テスト入力以外では発生しないことが望ましい）。

### 3.3. 鑑定護石のプログレッシブ入力
- **TS-UI-005: カスケード選択の強制 (Disabled制御)**:
  - **操作**: 新規登録を開き、レア度を選択せずにスキル選択ボタンを確認。
  - **期待値**: 第1スキル以降の各ボタン、およびスロット選択肢がすべて `disabled`（無効化）になっていること。
- **TS-UI-006: 上流変更時の自動リセット**:
  - **操作**: 
    1. レア度8、第1、第2スキル、スロットを選択済み。
    2. レア度を 7 に変更する。
  - **期待値**: 第1スキル〜第3スキル、およびスロットの選択状態がすべてクリア（未選択状態）になり、第1スキル以外が再び `disabled` に戻ること。
- **TS-UI-007: ダイアログ内シームレス遷移**:
  - **操作**: 「スキル選択」をクリック後、スキル名の1つをクリック。
  - **期待値**: ダイアログが閉じずに、そのスキルが取り得るレベル（Lv1, Lv2など）のボタン表示に切り替わり、レベル確定後に初めてダイアログが閉じること。
- **TS-UI-008: スリムカードへのレベル表示**:
  - **操作**: 護石を登録。
  - **期待値**: 画面上のカードにスキル名だけでなく、レベルが「（Lv2）」のように併記されていること。

## 2. ストレージ・統合検証 (Storage Integration)

### 2.1. データ圧縮 (zlib)
- **TS-ST-001: zlib圧縮と永続化**:
  - **入力**: 大量のダミーレコードを登録。
  - **期待値**: zlib圧縮され、base64文字列としてCookieに保存されること。読み込み時に正しく復元されること。

### 2.2. 同期ロジック
- **TS-ST-002: Cookie -> Supabase 同期**:
  - **入力**: 匿名で利用後、ログイン。
  - **期待値**: ローカル（Cookie）のデータがSupabaseにアップロードされ、以後クラウド優先で管理されること。

## 3. UI/UX・手動操作検証 (Manual Scenarios)

### 3.1. スキルピッカー
- **TS-UI-001: ピッカーの自動クローズ**:
  - **操作**: スキル名をクリック。
  - **期待値**: ダイアログが即座に閉じ、選択したスキルがフォームに反映されること。
- **TS-UI-002: お気に入り操作**:
  - **操作**: ダイアログ内で ⭐ ボタンをクリック。
  - **期待値**: ダイアログが**閉じず**に ⭐ 状態が切り替わり、リストの最上部にお気に入り項目がソートされること。

### 3.2. 全画面 Undo/Redo
- **TS-UI-003: 登録のUndo**:
  - **操作**: 武器を登録直後に「Undo」をクリック。
  - **期待値**: 登録した武器がリストから消え、状態が戻ること。
- **TS-UI-004: 強化適用のUndo**:
  - **操作**: スキルを武器に適用直後に「Undo」をクリック。
  - **期待値**: カウントが元に戻り、武器のスキルも適用前の状態に復元されること（データの不整合がないこと）。
