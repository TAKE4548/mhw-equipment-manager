# UI Specification: MHWs Equipment Manager

## 画面構造とナビゲーション
- **Portal (🏠)**: 入口画面。メトリクスとクイックアクセス。
- **所有巨戟アーティア一覧 (📦)**: 武器の登録・編集・管理。
- **鑑定護石管理 (📿)**: 護石の登録、検索、お気に入り管理。
- **スキル抽選結果 (⚔️)**: スキル抽選シードの管理。
- **復元強化厳選 (✨)**: 復元ボーナステーブルの進行管理。

## AIフレンドリー・コンポーネント設計 (標準仕様)
本プロジェクトのUI実装は、AIエージェントによるメンテナンス性を最大化するため、以下の **5-File Pattern** を厳格に遵守する。

| ファイル | 責務 | 命名規則例 |
| :--- | :--- | :--- |
| **State** | 状態初期化/リセットロジック | `state.py` |
| **Atoms** | 最小パーツ (バッジ、整形HTML) | `atoms.py` |
| **Dialogs** | `@st.dialog` ロジック | `dialogs.py` |
| **Form** | 入力・検索・登録用 Fragment | `form.py` |
| **List** | 一覧・グリッド用 Fragment | `list.py` |

### 物理分割のメリット
1. **トークン消費の極小化**: デザイン修正時に 15KB の巨大ファイルを読まず、2KB の `atoms.py` だけを読み取れば良くなる。
2. **サイドエフェクトの防止**: 入力フォームの修正が一覧表示のロジックを破壊するリスクを物理的に遮断する。

## レンダリングアーキテクチャ
高密度データを高速に描画し、入力ラグを最小化するための構成標準：

- **フラグメント隔離 (st.fragment)**:
  - ページ内の「入力エリア（Form）」と「表示エリア（List）」を別の `st.fragment` で分割する。
  - **メリット**: フォームへの 1 文字入力が、重いリスト描画の再計算を誘発しない。
- **2段構成・高密度マルチラインカード (v15)**: 
  - 1レコード64px固定。
  - **IDクラスター**: 武器種（上段・強調）と武器名（下段・控えめ）を縦に配置し、横幅を節約。
  - **アイコン・プレフィックス**: 各行の役割（🛡️シリーズ, 👥グループ, 🛠️生産, ✨復元）をアイコンで明示し、認知負荷を低減。
  - **一般構成**: 名前/種別(2段)、属性(中央)、強化タイプ(中央)、スキル(2段)、ボーナス(2段)。
  - **復元強化構成**: 名前/種別(2段)、属性(中央)、「生産/タイプ」(2段)、「復元前後」(2段)を配置。
  - **整列**: 右寄せカラム内のテキストは左寄せ（先頭アイコン揃え）。
  - **護石**: 既存のウルトラスリムスタイル（40px/1行）を維持。

## 詳細機能仕様
- **ビジュアル武器選択 (Visual Weapon Selector)**:
  - 階層型フィルタ（武器種→属性→スキル→ボーナス）による動的な絞り込み。
  - 3カラム・グリッドレイアウトによるスクロール量の削減。
- **プログレッシブ入力制御**:
  - 依存関係（例: レア度→第1スキル）がある場合、前段未入力時は後段を `disabled` にする。
  - 上流の変更時は下流を強制リセット（`state.py` の関数を呼び出す）。
- **用語の一元管理 (REQ-025)**:
  - UI文字列は `src/locales/ja.json` から `t()` 関数で取得する。
- **高密度マルチラインカード (v15)**:
  - **目的**: 武器種/名、属性、強化タイプ、スキルx2、ボーナスx2 の全情報を一覧可能にする。
  - **レイアウト構造 (標準)**:
    - [Col1: Badge] (y-center)
    - [Col2: ID Stack] -> Row1: Weapon Type (Bold) / Row2: Weapon Name (Small)
    - [Col3: Elem/Enh] -> Row1: Element / Row2: Enhancement Type (または両方中央)
    - [Col4: Skills] -> Row1: 🛡️ Skill1 / Row2: 👥 Skill2
    - [Col5: Bonuses] -> Row1: 🛠️ Prod. / Row2: ✨ Rest. (※右寄せ・アイコン揃え)
  - **レイアウト構造 (復元強化)**:
    - [Col1: Badge] (y-center)
    - [Col2: ID Stack] -> Row1: Type / Row2: Name
    - [Col3: Info Row] -> Row1: 🛠️ Prod. / Row2: 📋 Enh. Type
    - [Col4: Comparison] -> Row1: ✨ [Current] / Row2: ➡️ [Target]

## 誤操作防止（Safety First）パターン
- **ロック（保護）機能**:
  - 武器の操作メニューに「🔒 ロック/解除」トグルを配置する。
  - ロックされた武器には名前の横に 🔒 アイコンを表示し、視覚的に保護状態を明示する。
- **操作の制限 (Action Blocking)**:
  - ロックされたアイテムに対し、「編集」および「削除」ボタンをプログラム的に無効化（`disabled=True`）する。
  - 無効化されたアクションにマウスホバーした際、「ロックされているため操作できません」といった旨のツールチップを表示する。
  - この仕様は、従来の確認ダイアログ（2段階削除）よりも強力な保護手段として機能する。
