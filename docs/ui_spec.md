# UI Specification: MHWs Equipment Manager

## 画面構造とナビゲーション
- **Portal**:
    - **Home (🏠)**: アプリケーション全体の入口。武器・護石の登録サマリー（数値メトリクス）および主要機能へのクイックアクセスを表示。
  - **装備管理**:
    - **所有巨戟アーティア一覧 (📦)**: 所持武器の登録・編集・削除。
    - **鑑定護石管理 (📿)**: ランダム護石の登録、スキル・スロット検索、お気に入り管理。
  - **強化・抽選**:
    - **スキル抽選結果 (⚔️)**: スキル抽選シードのトラッキング管理。
    - **復元強化厳選 (✨)**: 武器個別の復元ボーナステーブル進行管理。
  - **その他**:
    - **分析統計 (📊)**: Altairを使用した統計分析ダッシュボード。旧Homeから移行。

## Lean Design 指針 (REQ-007)
アプリケーション全体の視覚的密度を高め、操作効率を最大化するため以下の設計指針を適用します：

- **統合型リーン・ヘッダー**: 
    - ページタイトル、説明文、および Undo/Redo 操作系を 1 つのコンパクトなエリアに集約。
    - Undo/Redo はタイトルの右側にアイコンボタンとして配置し、画面中央の大きなスペースを占有させない。
- **高密度レイアウト (Localized Density)**:
    - **Roomy Layer & Data Layer**: ページ全体を「Roomy（標準余白）」、データリストを「Tight（2px）」として層分離。
    - `st.divider()` を廃止し、`.lean-sep` (`margin: 1.5rem 0`) で区切りを表現。
    - 主要ウィジェット（エキスパンダー、ボタン）は `1rem` の標準余白を維持し、誤操作を防止。
    - データリスト内のみ、CSS により垂直方向の隙間を `2px` に圧縮。
- **省スペース型コントロール**:
    - エキスパンダー内の余白を削減。
    - フォーム内のラベル表示を最適化。


- **共通設定サイドバー**:
  - 全ページで同一の「ストレージ設定」（ローカル保存 / クラウド同期）とログイン機能にアクセス可能。

## コンポーネント役割と表示ロジック
- **統合型登録フォーム**:
  - 新規スキル抽選結果の登録は、メイン画面トップのエキスパンダー内で行う（ページ遷移を最小化）。

- **ビジュアル武器選択 (Visual Weapon Selector / REQ-024)**:
  - 抽選結果の登録時、対象の武器を素早く特定するための高効率選別UI。
  - **階層型フィルタ (Hierarchical Filtering)**:
    - 「1. 武器種」⇒「2. 属性」⇒「3. スキル」⇒「4. ボーナス」の順で段階的に絞り込みを行う。
    - 各フィルタは、前の条件で該当する武器が存在する選択肢のみを動的に提示する（無駄な空振りを防ぐ）。
  - **3カラム・グリッドレイアウト**: 
    - 抽出された武器を 3 カラム形式で並列表示し、縦方向のスクロール量を 1/3 に圧縮。
    - 各カードは HUD v14 準拠の `render_selectable_card` を使用。
  - **リアルタイム反映**:
    - フィルタの変更は `st.fragment` を介して即座にグリッドに反映。
    - 武器カード内の「❯/✔」ボタンをクリックすることで選択を確定し、登録フォームに遷移。
  - **視覚的フィードバック**: 選択状態の武器カードはゴールド発光ボーダー（`v12-selected`）で強調。

- **強化前後の比較バー (HUD Micro-Diff / v14)**: 
  - 5つの枠（ボーナス）を一塊のセットとして捉え、現状と抽選結果のボーナスの推移（before ❯❯ after）を、テーブルではなく「色の帯（バー）」と「輝度の差」を模した 1 行の HTML 要素で表現します。
  - 各項目を等幅グリッドで配置し、文字数に関わらない完璧な上下整列を実現します。
  - 変化がある要素を強調（ゴールド色への変化による Diff 表示）。

- **プログレッシブ入力制御 (Progressive Disclosure)**:
  - フォームにおいて、入力に前後関係や依存関係（例: レア度→第1スキル→第2スキル、あるいは枠1→枠2）がある場合、前段の入力が完了するまで後段の要素を `disabled` (無効化) 状態とし、左から右へ自然と順番に誘導する設計を取り入れます。
  - **依存関係のリセット**: レア度や上流のスキルが変更された場合、整合性を保つためそれ以降の全項目の選択状態を強制的にリセット（初期化）します。これにより常に「正しい組み合わせ」のみが入力される状態を保証します。

- **抽選結果一覧の垂直整列 (REQ-026)**:
  - リスト内の「残り回数」を常に同じ位置に表示し、かつスキル名の表示領域を最大化するため、以下の順序で配置します。
  - **整列構成**:
    - Identity (武器種/要素): 180px (固定)
    - Metric (残り回数): 100px (固定)
    - Spec (スキル詳細): 余り領域 (可変/Flex 1)
  - これにより、全行で「残り回数」が開始される 180px 地点、および「スキル詳細」が開始される 280px 地点が水平方向に完全に一致します。
  - 文字列が長い場合は、カードの右端まで可能な限り表示されます。

## パフォーマンス・描画アーキテクチャ
高密度なデータを高速に描画し、入力ラグを最小化するため、以下の構成を標準とします。

- **フラグメント隔離 (st.fragment)**:
  - 「登録フォーム」と「一覧表示リスト」は必ず別の `st.fragment` 単位で隔離します。これにより、フォームへの1文字入力（Session State更新）が、重いリスト描画の再トリガーにならないように制御します。
  - **遵守事項（不整合防止）**:
    1. **グローバル同期の強制**: リスト内での削除や更新によってページ全体の再描画が必要な場合（例：ヘッダーの統計値が変わるなど）、明示的な `st.rerun()` を呼び出してページ全体の整合性を保つこと。
    2. **データの共通参照**: フラグメント外で読み込んだデータを引数として渡す際、そのデータが最新であることを保証するため、I/O層（load_* 関数）のキャッシュクリアタイミングに注意すること。
    3. **Session State の競合回避**: フラグメント内でのみ使用する一時的な状態（例：検索フィルタの途中経過）は、キーにユニークなプレフィックスを付けるか、意図しないリセットが起きないよう設計すること。

### 2.1 新規コンポーネント: Portal (Home)
- **役割**: ユーザーが最初に見る画面。主要機能へのショートカットと、現在の全体ステータスを提供する。
- **データ参照**: `SessionState` から現在の武器数、護石数等を取得。
- **UI 構造**: 
  - **シンプルボタン**: 複雑な Overlay を排除し、標準 `st.button` を CSS で大判化して使用。
  - **ミニマルデザイン**: ボタン内を「アイコン ＋ タイトル」に限定し、説明文（description）を削除することで視覚的整合性を確保。
  - **視覚的フィードバック**: 未実装機能はグレースケールと 20% 透過を適用し、ホバーエフェクトを無効化することで deactive 状態を明示。
  - **統計**: `st.metric` による数値のみのサマリー（3 列構成）。

- **事前データ加工 (Pre-render Preparation)**:
  - 描画ループ（`for` 文）内での `merge`, `normalize`, `load_data` 呼び出しを禁止します。
  - ループに入る前に必要なデータをすべて結合・加工済みの DataFrame として用意し、ループ内では単純な参照のみを行う構成にします。
- **ロジック層のキャッシュ化**:
  - `master_data.json` 等の静的データ読み込み、および正規表現を用いた略称変換（`get_abbr_item` 等）には必ず適したキャッシュ（`st.cache_data` または `lru_cache`）を適用します。

## 表示用語の一元管理 (REQ-025)
アプリケーション内の主要な UI 文字列は、`src/locales/ja.json` で一元管理されています。
- 文言の変更や調整は、コードを直接編集するのではなく、この JSON ファイルを修正することで全画面に反映されます。
- 文字列の取得には `src.utils.i18n.t` 関数を使用します。
- 将来的な多言語化が必要な場合は、`src/utils/i18n.py` のローダーを拡張することで対応可能です。

## 表示用語の統一 (Terminology)
- **ボーナス名称**: 
  - 生産: `基礎攻撃力増強`, `会心率増強`
  - 復元（厳選対象）: `基礎攻撃力強化`, `会心率強化`, `属性強化`, `切れ味強化`, `装填強化`
- **レベル表記**: 
  - ローマ数字（`Ⅰ`, `Ⅱ`, `Ⅲ`）および `EX`, `無印` を使用。
