# Feature Specification: MHWs Equipment Manager

**Feature Branch**: `001-mhw-eq-manager`  
**Created**: 2026-03-20  
**Status**: Draft  
**Input**: User description: "Create MHWs equipment manager MVP focusing on Kyogeki Artia weapon skill upgrade tables"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Register Skill Upgrades (Priority: P1)

ユーザーは、武器種、属性、シリーズスキル、グループスキル、および強化回数を指定することで、巨戟アーティア武器用の新しいスキル強化レコードを登録できます。

**Why this priority**: 強化回数のトラッキングを行う前に、まずは対象となるスキル強化を登録する必要があるため、この機能が中核機能（コア）となります。

**Independent Test**: 登録フォームに必要事項を入力して送信し、新しいレコードが正しく初期回数とともにリスト画面に表示されるかを確認することで単独テスト可能です。

**Acceptance Scenarios**:

1. **Given** アプリケーションが開いている状態で、**When** ユーザーが武器種、属性、シリーズスキル、グループスキル、強化回数を入力して送信すると、**Then** アクティブな強化リストに新しいレコードが追加される。

---

### User Story 2 - Execute Skill Upgrades (Priority: P1)

ユーザーは、リストから登録済みのスキル強化を選んで実行（すべて完了）でき、その結果、**すべてのアクティブなレコード**の残り強化回数が減少します（錬成シードの全体進行を反映）。

**Why this priority**: ゲームプレイ中に進捗をトラッキングするための主要な操作機能です。

**Independent Test**: 既存のレコードを選択し、強化を実行して、回数が正常に減少するかを観察することで単独テスト可能です。

**Acceptance Scenarios**:

1. **Given** 回数が5回残っている登録済み強化Aと、回数が3回残っている強化Bが存在する場合、**When** ユーザーがAを対象に「Execute」を一括実行すると、**Then** Aの残り回数が0回になり削除され、同時にBの残り回数も2回（3 - (-5)ではありませんが、Aの元々の回数分である5回をマイナスするため、3-5=0未満のため自動的に0になり）削除される。

---

### User Story 3 - Auto-remove Completed Upgrades (Priority: P2)

実行後、スキル強化レコードの残り回数が0回以下に達すると、自動的にアクティブなリストから削除されます。

**Why this priority**: UIをクリーンな状態に保ち、未完了のタスクだけに集中できるようにするためです。

**Independent Test**: 対象レコードに対して最後の強化（残り1回の状態から1回実行）を実行し、そのレコードがリスト画面から消えるか確認することで単独テスト可能です。

**Acceptance Scenarios**:

1. **Given** 回数が1回残っている登録済み強化が存在する場合、**When** ユーザーが1回分実行すると、**Then** 残り回数が0回になり、リストからそのレコードが削除される。

---

### User Story 4 - Undo/Redo Actions (Priority: P2)

ユーザーは、直前の操作（登録や実行）を取り消し（Undo）たり、取り消しをキャンセル（Redo）したりすることで、ゲームプレイ中の誤ったクリック操作を容易に修正できます。

**Why this priority**: プレイ中のトラッキングでは間違った操作をしてしまうことが多いため、ユーザー体験（UX）向上において非常に重要です。

**Independent Test**: 何らかのアクション（実行など）を行った後、Undoボタンをクリックして状態が元に戻るかを確認し、次にRedoボタンをクリックして再度アクションが適用されるかを確認することで単独テスト可能です。

**Acceptance Scenarios**:

1. **Given** ユーザーがたった今強化を実行した状態で、**When** Undoをクリックすると、**Then** 強化回数が実行前の値に復元される。
2. **Given** ユーザーがたった今アクションを取り消した状態で、**When** Redoをクリックすると、**Then** 取り消されたアクションが再度適用される。

### Edge Cases

- ユーザーが、既存の「武器種/属性/シリーズスキル/グループスキル」の組み合わせと全く同じレコードを追加しようとした場合どうなるか？（別レコードとして保持するか、回数を合算するか？）
- Undo / Redo の履歴はどこまで遡れるようにするか？（メモリ節約のため10回までとするか、など）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ユーザーは特定の属性（武器種、属性、シリーズスキル、グループスキル）を持ったスキル強化回数を登録できること。
- **FR-002**: ユーザーは、既存のスキル強化レコードの回数を減らす（実行する）ことができること。
- **FR-003**: システムは、残り回数が0回以下になったレコードを自動的にリストから削除すること。
- **FR-004**: システムは、登録および実行アクションの Undo（取り消し）と Redo（やり直し）をサポートすること。
- [x] **FR-005**: システムは、ユーザー登録なしで直ちに利用できるよう、ブラウザのローカルCookie領域にデータを自動永続化し、StreamlitのSandboxを回避した書き込みを行うこと。
- [x] **FR-006**: システムは、ユーザー登録・ログインを行うことで、クラウド（Supabase）を介したデバイス間でのデータ同期とバックアップ機能を提供すること。
- [x] **FR-007**: 登録フォーム上の「シリーズスキル」と「グループスキル」は、`src/data/master_data.json` で定義されたマスタデータを元に選択肢として表示されること。
- [x] **FR-008**: UIレイアウトは、Undo/Redoと「新規登録」ボタンを同列のツールバーに配置し、テーブルビューは残り回数昇順でソートされ、各フィールド（グループスキル含む）は独立した固定幅カラムとして表示され位置ズレを起こさないこと。
- [x] **FR-009**: スキルマスタ（JSON）は、シリーズ部品名/グループ名と、その発動スキル名のペアで構造化されていること。登録・一覧の各画面はこの構造を参照し、効果名を表示に反映させること。

### Key Entities

- **Skill Upgrade Record (スキル強化レコード)**: 対象となるスキル強化のデータ行。（属性：武器種、属性、シリーズスキル、グループスキル、残回数）。
- **Action History (アクション履歴)**: Undo/Redo機能を実現するための過去のアクション履歴スタック。

## Success Criteria *(mandatory)*

### User Story 5 - Manage Kyogeki Artia Equipment (Priority: P1)
ユーザーは、巨戟アーティア武器固有のパラメータ（生産ボーナス、巨戟強化、復元ボーナス）を所持武器ごとに登録・管理できます。

### User Story 6 - 強化厳選登録 & Table Sync (Priority: P1)
ユーザーは、未来の強化抽選結果を「抽選結果」として複数登録でき、特定の抽選結果を「強化実施」することで武器のステータスを更新し、同時に共通テーブルを進行（他の抽選結果をスキップ）させることができます。

## UI/UX Standards (Phase 2.11)
視認性と操作性を両立するため、以下のUI標準を適用しています。

- **属性別カラーバッジ**: 
  - 属性名称のみ（火, 水, 雷...）の単体バッジ。MHカラー背景。
- **超スリム・シングル行カード**: 
  - 1レコード1行。全画面で「ウルトラスリム」なカード形式を採用。
- **ビジュアル武器選択 (Visual Selection)**: 
  - 抽選結果の登録や付与の際、詳細情報を含んだ「ミニカード一覧」から視覚的に武器を選択可能。
- **強化前後の比較テーブル (復元強化厳選)**: 
  - HTMLベースのグリッドにより、現在と抽選結果のボーナス枠を完璧に縦列整列して表示。
  - 変化があるスロットを強調（Diff表示）。
- **共通設定サイドバー**:
  - 全ページで同一の「ストレージ設定」（ローカル保存 / クラウド同期）とログイン機能にアクセス可能。
- **統合型登録フォーム**:
  - 新規スキル抽選結果の登録は、メイン画面トップのエキスパンダー内で行う（ページ遷移を最小化）。

## Terminology & Navigation
- **ナビゲーション構造**:
  - **Home (🏠)**: 全体の統計分析（ダッシュボード）。武器種別のシェアや属性・スキルの分布を可視化。
  - **所有巨戟アーティア一覧 (📦)**: 所持武器の現状管理。
  - **スキル抽選結果 (⚔️)**: スキルの付与進行と新規登録。
  - **復元強化厳選 (✨)**: 未来の復元テーブルの管理とシミュレーション。
- **ボーナス名称**: 
  - 生産: `基礎攻撃力増強`, `会心率増強`
  - 復元（厳選対象）: `基礎攻撃力強化`, `会心率強化`, `属性強化`, `切れ味強化`, `装填強化`
- **レベル表記**: 
  - ローマ数字（`Ⅰ`, `Ⅱ`, `Ⅲ`）および `EX`, `無印` を使用。

## Phase 2 Requirements
- **FR-010**: 復元ボーナスの「同種同一レベル（強化済み）」の重複は2つまでとする制約バリデーション。
- **FR-011**: 復元テーブルの進行時、残り回数が0以下になった他の全目標を自動削除（スキップ）する機能。
- **FR-012**: 操作ミス防止のための Undo / Redo 機能（最大5件まで）。
- **FR-013**: 所有武器データを統計的に可視化し、武器種・属性・スキルごとの偏りを分析できるダッシュボード機能。
- **FR-014**: 頻繁に使用するスキルをお気に入り（⭐）として登録し、UI上の選択リストで最優先表示する機能。

## Data Model (Cloud Extensions)
### Table: `favorites`
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID | Primary Key |
| `user_id` | UUID | Auth user association |
| `favorite_type` | TEXT | 'series' or 'group' |
| `skill_value` | TEXT | Skill name or ID |
| `created_at` | TIMESTAMPTZ | Creation timestamp |

## Success Criteria (Phase 2)
- **SC-004**: 武器の「適用」ボタン一つで、武器DBの更新、全トラッカーの減算、スキップ分の削除が3秒以内に完了すること。
- **SC-005**: 複雑な進行（スキップ）が発生しても、Undoによって全シートの状態が整合性を保ったまま完全に復元されること。
- **SC-006**: お気に入りスキルが、新規登録および強化登録のすべてのセレクトボックスで最上位に配置されること。
