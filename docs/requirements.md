# 要求事項 (Requirements) - 実装状況

本アプリの開発において定義された要求事項と、その実装状況です。

- [x] **各ページでのレコードのソート・フィルタ機能**
  - [x] 各項目で AND フィルタ（複数選択可能）を適用。
  - [x] 生産ボーナス・復元ボーナスは組合せ（順不同）で検索可能。
- [x] **レスポンシブ・スリムデザインの導入**
  - [x] スマホ閲覧時の一望性を高めるため、極限まで余白を削ったスリムカードデザインを採用。
- [x] **ログイン状態の永続化**
  - [x] `streamlit-cookies-controller` を使用し、ブラウザを閉じてもログインセッションが維持されるよう実装。
- [x] **統計分析ダッシュボード (Home)**
  - [x] 所有武器の分布（武器種・属性・スキル）を可視化する Altair グラフを導入。
  - [x] 武器種ごとのドリルダウン分析機能を実装。
- [x] **ダイアログ形式のスキルピッカー (st.dialog)**
  - [x] スキル名選択時は 1クリックで自動クローズし、操作ステップを最小化。
  - [x] お気に入り（⭐）操作時は `st.fragment` によりメニューを維持し、連続操作を可能に。
- [x] **レスポンシブな横並びレイアウト**
  - [x] 「シリーズスキル」と「グループスキル」を画面幅に応じて横並び（2カラム）または縦並びに自動調整。
- [x] **分析ダッシュボードの整理**
  - [x] 「分析ダッシュボード」を「Home」にリネームし、ナビゲーションの順序を最適化。
- [x] **スリムデザインの統一**
  - [x] 各ページの各レコードのカードデザイン方針を、所有武器一覧のスタイル（超スリム）に統一。
- [/] **システムの安定化とクリーンアップ (Ongoing)**
  - [ ] 不要コード (`gsheets_manager_old.py` 等) の削除
  - [ ] データ永続化ロジック (zlib圧縮Cookie + Supabase) の仕様化
  - [ ] 自動テストによるロジック担保 (User指示待ち)
- [x] **鑑定護石（ランダム護石）管理機能 (Completed)**
  - [x] Rarity (レア度) ごとのスキル・スロットバリデーション機能
  - [x] スキルグループ・レア度パターン管理用の外部マスタデータ化 (`talisman_master.json`)
  - [x] **プログレッシブ・カスケード制御の導入**:
    - [x] レア度→スキル1→スキル2→スキル3→スロットの順で入力を強制し、未入力時は後段を無効化するUI。
    - [x] 「スキル選択」→「レベル選択」を同一ダイアログで完結させる2ステップUIの実装。
    - [x] **依存関係のリセット**: レア度や上流スキルが変更された際、それ以降の下流データを自動クリアする挙動。
  - [x] スリムデザインを採用した護石一覧・一括管理画面の実装

- [x] **強化厳選画面のリファクタリング (Completed)**
  - [x] **Visual Band 遷移 UI**: 比較表の「現/目」ラベルを廃止し、左端のカラーバーと背景輝度、および5スロット完全整列のグリッドによる直感的な推移表現。
  - [x] **武器選択カードの情報拡充**: シリーズスキルに加え、グループスキルと生産ボーナス（3枠）をカード内に表示。
  - [x] **厳選リストのフィルタ・ソート**: 登録済みトラッキング一覧に対し、武器種・属性・残り回数等での絞り込みと並び替え機能の実装。

- [ ] **システムのパフォーマンス最適化 (Planned)**
  - [ ] **キャッシングの導入**: `@st.cache_data` (I/O) および `functools.lru_cache` (ロジック) を適切に適用。
  - [ ] **描画範囲の隔離**: `st.fragment` を用い、入力フォームと一覧リストの再描画スコープを分離。
  - [ ] **データ取得効率の改善**: ループ内での I/O 呼び出しを排除し、描画前にデータを一括ロードする構造へのリファクタリング。
