# System Design: REQ-009 ページ構成とナビゲーションの再設計

## 1. 概要
アプリケーションの入口を「分析ダッシュボード」から「クイックポータル」に変更し、ナビゲーションを論理的なワークフローに基づいてグルーピングすることで、ユーザー体験（UX）を向上させる。

## 2. アーキテクチャとデータフロー
### 2.1 新規コンポーネント: Portal (Home)
- **役割**: ユーザーが最初に見る画面。主要機能へのショートカットと、現在の全体ステータスを提供する。
- **データ参照**: `SessionState` から現在の武器数、護石数等を取得。
- **UI 構造**: 外部カード（Markdown）を排除し、`st.button` を大判化（CSS）して全体をクリッカブルにする。円グラフは廃止。

### 2.2 ナビゲーション構造 (st.navigation)
Streamlit 1.55+ のセクション機能を利用し、以下のように再構成する。

| セクション | ページ | 説明 |
| :--- | :--- | :--- |
| **Portal** | ホーム | クイックアクセス、最近の操作、統計サマリー |
| **装備管理** | 所持武器台帳 | 所有武器の登録・一覧・編集 |
| | 鑑定護石管理 | 護石の登録・検索 |
| **強化・抽選** | スキル抽選管理 | スキル抽選テーブルのトラッキング |
| | 復元強化厳選 | 復元ボーナスの厳選・進行 |
| **その他** | 分析統計 | 武器・属性分布の可視化 |

## 3. 既存コンポーネントへの影響
- **app.py**: ナビゲーション定義を全面的に書き換える。
- **pages/4_analytics_dashboard.py**: ファイル内部のタイトルやアイコン設定を「Home」から「分析統計」に適した内容に変更。
- **pages/equipment_box.py**: 直接的な影響はないが、ポータルからのリンク受け入れ等の拡張性を考慮。

## 4. タスク分解
1. [ ] `pages/0_home.py` の新規作成（ポータル画面の実装）
2. [ ] `app.py` の修正（ナビゲーションのグルーピング化）
3. [ ] `pages/4_analytics_dashboard.py` の微調整（タイトル変更等）
4. [ ] 各ページ間の整合性テスト（ナビゲーション移動の検証）

## 5. テスト要件
### 5.1 単体テスト (Manual/Browser)
- ホームの各ボタンが正しいページに遷移すること。
- サイドバーにセクション名（「装備管理」など）が表示され、折り畳み/展開が機能すること。
- デフォルトページがホームになっていること。

## 6. Acceptance Criteria (受入基準)
- 主要なワークフロー（登録→抽選→厳選）に最適化されたページ階層が定義されている。
- Home 画面が主要アクションへの「ポータル」として機能している。
- ナビゲーションの構成が直感的である。
