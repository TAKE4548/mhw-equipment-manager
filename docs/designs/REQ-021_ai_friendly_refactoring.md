# 設計書: AIフレンドリーなコンポーネント指向リファクタリング (REQ-021)

## 1. 背景と目的
Streamlit アプリケーションのページファイルが巨大化（15KB超）すると、AIエージェントが修正を行う際に大量のトークンを消費し、文脈の制限による誤修正やデグレードのリスクが高まる。
本設計は、ページを役割ごとに物理的な最小単位に分割することで、AIエージェントが必要最小限のファイルだけを読み取り、安全かつ低コストでメンテナンスを行える「AIフレンドリー構造」を確立することを目的とする。

## 2. 標準ディレクトリ構造
全ての主要ページは、以下のディレクトリ構成に従って `src/components/{page_name}/` 配下に分割される。

| ファイル名 | 役割 | 内容 |
| :--- | :--- | :--- |
| `state.py` | 状態管理 | `st.session_state` の初期化、リセット、同期ロジック。 |
| `atoms.py` | 最小表示パーツ | バッジの色、スロットアイコンの整形など、UIの最小要素。 |
| `dialogs.py` | インタラクション | `@st.dialog` を用いた複雑なポップアップロジック。 |
| `form.py` | 入力/登録 | 検索フィルタ、登録フォームなどの入力エリア。 |
| `list.py` | 一覧表示 | データの一覧、グリッド表示、カードのレンダリング。 |

## 3. 実績 (適用済みページ)
以下のページに対して本設計を適用済みである。

1. **鑑定護石管理** (`pages/5_talismans.py`)
2. **復元強化登録** (`pages/reinforcement_registration.py`)
3. **装備ボックス** (`pages/equipment_box.py`)
4. **スキル抽選結果** (`pages/0_skill_lottery.py`)

## 4. ページ本体 (`pages/*.py`) の責務
ページ本体は、上記のコンポーネントをインポートして配置するだけの「オーケストレータ」に徹する。
- ページ設定 (`st.set_page_config`)
- サイドバーの描画
- セッション状態の初期化
- コンポーネントの呼び出し

## 5. UI ポリシー (v14基準)
- **カラーパレット**: 武器属性やアクションボタンに一貫したカラー（属性バッジ等）を使用する。
- **高密度レイアウト**: `.v12-dense-list` CSS クラスをカード一覧に適用し、視認性と密度を両立する。
- **AI 修正プロンプト**: デザイン修正を依頼する場合、AI は `atoms.py` を優先的に読み込み、修正対象を特定する。
