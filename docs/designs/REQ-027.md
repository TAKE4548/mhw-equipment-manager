# Design: REQ-027 リストカードの2行表示対応

## Architecture
- `src/components/cards.py` の `inject_card_css` および `_render_v14_tag_body` を v15 (Multi-line) 仕様に拡張する。
- 既存の `render_slim_card` および `render_selectable_card` は引数のシグネチャを維持しつつ、内部的な HTML/CSS 構造を 2 行表示に切り替える。
- `CARD_ACTION_RATIO` (11.5:1) を維持し、Streamlit の `st.columns` による外部レイアウトの一貫性を担保する。

## Data Flow
1. `list.py` または `tables.py` が `render_slim_card` を呼び出す。
2. 内部で `_render_v14_tag_body` が呼び出され、新しいグリッド構成の HTML を生成する。
3. `st.markdown(unsafe_allow_html=True)` で描画される。
4. CSS クラス `v15-multi` が 64px の高さを強制し、コンテンツを 2 行に配置する。

## Testing Requirements
- **Visual Consistency**: `equipment_box`, `skill_lottery`, `reinforcement_registration` の各ページで、右端の「⋮」ボタンが正しく垂直中央（あるいは意図した位置）に揃っていること。
- **Text Overflow**: 非常に長いスキル名（例: 「冰龍の神秘 + 真・会心撃【属性】」等）が省略されずに表示されるか、あるいは適切な 2 行目で表示されていること。
- **Response**: ウィンドウ幅を縮小した際に、レイアウトが崩れずにレスポンシブに追従すること。
