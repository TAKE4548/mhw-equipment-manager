# System Design: REQ-024 武器選択UIの刷新 (Visual Weapon Selector)

## 概要
「復元強化厳選」ページにおける武器登録フローを改善するため、従来の「エキスパンダー+単一リスト」を廃止し、ユーザーの検索メンタルモデル（武器種→属性→スキル→ボーナス）に最適化された「階層フィルタ型選別UI」を導入します。

## 設計方針
1. **段階的開示 (Progressive Disclosure)**:
    - フィルタを「武器種」⇒「属性」⇒「スキル」⇒「ボーナス」の順に提示。
    - 前のステップが完了するまで次のフィルタを無効化（または非表示）にし、ユーザーの認知負荷を軽減します。
2. **リアルタイム・高密度グリッド**:
    - フィルタ操作が即座に反映される 3 カラム構成のカードリストを採用。
    - `st.fragment` を活用し、フィルタ操作時の「画面全体のガクつき」を抑制します。
3. **HUD v14 準拠のビジュアル**:
    - [11.5 : 1] の `CARD_ACTION_RATIO` は維持しつつ、グリッド表示により情報の密度を最大化します。

## コンポーネント構成

### 1. Hierarchical Filter Bar
- `st.columns` を用いて、左から右へ流れるように配置。
- 各セレクトボックスは、前のフィルタで絞り込まれた結果に基づく選択肢のみを表示します。

### 2. Result Grid (Responsive CSS Grid)
- 抽出結果の表示には `st.columns` ではなく、カスタム CSS による **CSS Grid** を採用します。
- `grid-template-columns: repeat(auto-fill, minmax(320px, 1fr))` を指定することで、画面幅に応じて自動的に 1〜3 カラムに変化し、各カードの視認性を保ちます。
- 武器カードには、既存の `render_selectable_card` を使用します。

## データフロー

1. `load_equipment` で全武器データを取得。
2. フィルタ状態 (`st.session_state`) に基づき、`src.logic.equipment_box.filter_equipment` でデータをフィルタリング。
3. 抽出された DataFrame を 3 カラムに分配して描画。
4. 選択（❯/✔）ボタン押下時、`tracker_reg_w_id` を更新して `st.rerun()`。

## 画面イメージ（概念）

```text
[ 武器種 ∨ ] -> [ 属性 ∨ ] -> [ スキル ∨ ] -> [ ボーナス ∨ ]
----------------------------------------------------------
[ 武器カード1 ] [ 武器カード2 ] [ 武器カード3 ]
[ 武器カード4 ] ...
```

## 影響範囲
- `pages/reinforcement_registration.py`: 登録セクションのUI実装を全面的に書き換えます。
- `src/components/cards.py`: 既存の CSS は流用しますが、グリッド表示用の補正が必要な場合があります。

## 検証プラン
- **単体テスト**: `filter_equipment` が期待通りに階層的な絞り込みを行えるか。
- **UIテスト (Manual)**:
    - フィルタ操作時にリストが即座に更新されるか。
    - グリッド表示が崩れず、垂直方向の整列が保たれているか。
    - 選択した武器が正しく登録フォームに引き継がれるか。
