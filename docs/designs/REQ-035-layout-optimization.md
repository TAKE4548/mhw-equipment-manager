# Design Document: REQ-035 Layout Optimization (v15 HUD)

## 1. 概要
複合アイコンの導入により、従来の「属性アイコン」用スペースが重複していた課題を解決し、情報密度の高いレイアウトへ最適化する。

## 2. 変更内容
### 2.1 Cluster システムの統合
- **Cluster 1 (Anchor)**: 複合アイコン（武器+属性）のみを表示。属性値はID Clusterへ移動。
- **Cluster 2 (Spec)**: 廃止。

### 2.2 ID Cluster の 2 行化
- **上段**: 固有名がある場合は「武器名」、ない場合は「武器種」。
- **下段**: 「属性名 + 属性値」（例：雷 510）。「属性」の文字は省き、アイコンと補完し合う。

### 2.3 カラム比率の再定義
| カラム | 旧幅 (v15) | 新幅 (v15 Optimized) | 変更理由 |
| :--- | :--- | :--- | :--- |
| `v15-col-anchor` | 42px | 46px | アイコンサイズに合わせた最適化 |
| `v15-col-spec` | 58px | - | ID Cluster へ統合し廃止 |
| `v15-col-id` | 160px | 160px | 属性情報を取り込むため幅を維持しつつ2行化 |
| `v15-col-skills` | 130px | 200px | スペースの全還元による見切れ防止 |
| `v15-col-count` | 90px | 80px | 最適化 |

## 3. トレードオフと制約 (Trade-offs and Constraints)
- **武器名の見切れ**: `v15-col-id` を 160px から 140px に縮小するため、極端に長い武器名（「ムフェト・ジーヴァの〜」等）が以前より早く見切れる可能性がある。
- **護石（Talisman）との整合性**: 護石は複合アイコンを使用しないため、`is_long` モードでは従来の `v15-col-spec` 構造を維持または別途調整し、レイアウト崩れを防ぐ必要がある。

## 4. UX デザイナーによるレビュー
- スキル名の視認性向上を最優先し、武器名の末尾が多少見切れることは許容範囲とする。
- 属性値ラベルから「属性」の文字を除き、数値または属性名のみにすることで垂直方向の圧迫感を軽減する。
