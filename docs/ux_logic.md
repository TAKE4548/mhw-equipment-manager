# UX Logics: MHW Equipment Manager

## REQ-072: 強化厳選における武器検索フィルタの拡張

### 1. 情報の優先順位とセクショニング
武器選択における検索ニーズを以下の3段階に整理し、3カラムUIにマッピングする。

- **Primary (基本属性)**: `武器名`, `武器種`, `属性`
  - ユーザーがまず「大剣の火属性で」といった大枠を決定する際に使用。
- **Secondary (ビルド要件)**: `シリーズスキル`, `グループスキル`, `巨戟強化`
  - 武器種と属性が決まった後、特定の「スキル（例：力自慢）」を持つ個体を探す際に使用。
- **Tertiary (強化・選定状態)**: `現ボーナス（種類）`, `並び替え`
  - 既に「攻撃強化」がついている個体や、特定の強化済みのものを探す際に使用。

### 2. 操作動線の設計
1. **expander の展開**: 高密度なフィルタを常に表示せず、必要なときだけ「開く」動作で集中力を高める。
2. **マルチセレクトによる柔軟な絞り込み**: シリーズ/グループスキルは数が多いため、文字入力補完が可能なマルチセレクトを採用し「これかこれかこれ」という自由な検索を許容する。
3. **ボーナスフィルタのAND検索ロジック**:
    - ユーザーは「攻撃Ⅲを2つ持っている武器」ではなく、「攻撃Ⅲと会心Ⅲの両方を持っている武器」を探すニーズが強いため、マルチセレクトで選択された**すべての要素**が復元ボーナス枠（1-5枠）の中に含まれているかを判定するAND検索を採用する。
    - 種類だけでなく「レベル（Ⅱ, Ⅲ, EX）」まで含めた完全一致によって、素材としての価値を正確にフィルタリング可能にする。


### 3. モバイル対応
- 3カラム構成はデスクトップでの一覧性を優先。
- モバイル表示時（Streamlit標準動作）は各カラムが縦に並ぶが、情報の優先順位（Primary -> Tertiary）の順で並ぶため、スクロールダウンによる自然な絞り込みが維持される。
