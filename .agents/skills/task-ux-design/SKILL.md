# 🎨 UX/UI Design Workflow
### Phase 0: パターンの抽出と合意
- 要求に対し、異なるアプローチの解決案を複数（A/B案など）提示せよ。
- **ユーザーが方針に同意するまで、次のPhaseに進んではならない。**
### Phase 1: 体験の論理構成
- 情報の優先順位と操作動線を定義し、`docs/ux_logic.md` 等に根拠を記録せよ。
### Phase 2: インターフェースの具体化
- `docs/ui_spec.md` を更新し、要素の配置や反応を記述せよ。
### Phase 3: 整合性検証
- 仕様がアーキテクトの定義するデータモデルと矛盾しないか確認せよ。
