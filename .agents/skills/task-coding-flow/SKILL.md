# 🛠️ Implementation Workflow

### Phase 0: 仕様の同期
- `docs/` の最新仕様とアーキテクトによる「実装プラン」を熟読し、修正箇所を完全に特定せよ。

### Phase 1: 環境準備
- 実装用のGitブランチを作成し、必要に応じて仮想環境や依存関係を確認せよ。

### Phase 2: テストの先行作成 (Test-First)
- ロジック修正前にテストコードを作成するか、既存のテストを更新して期待される挙動を定義せよ。

### Phase 3: 実装と自律検証
- `design_system.md` のスタイルと `domain_rules.md` のロジックを厳守せよ。
- `browser_subagent` 等を活用し、実際の画面で意図通り動くか自律的に確認せよ。
- **検証報告**: 完了時、どのようなテストを行い、どのような結果（成功ログ等）が得られたか具体的に記述せよ。

### Phase 4: 高品質なコミットと完了報告
- 以下のプレフィックスを用いた、粒度の細かいコミットを実行せよ。
  - `feat:` (新機能), `fix:` (バグ修正), `refactor:` (コード整理), `test:` (テスト追加), `docs:` (文書更新), `chore:` (雑務)
- `docs/workflow/tasks.md` を更新し、完了を宣言せよ。
