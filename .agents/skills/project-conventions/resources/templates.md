# Agent Shared Templates (v1.0)

This file is the Single Source of Truth (SSoT) defining the standard formats for artifacts generated by AntiGravity agents.

---

## 2. Implementation Plan Template

Standard section structure for `implementation_plan.md`.

### Mandatory Sections
1. **Goal Description**: Background and purpose of the change (in Japanese).
2. **User Review Required**: Changes requiring explicit approval (Use [!IMPORTANT] alerts).
3. **Proposed Changes**: Detailed changes per file.
4. **Trade-off Disclosure**: Sides effects, constraints, or downsides (Use [!CAUTION] or [!WARNING] alerts).
5. **Open Questions**: Unresolved points.
6. **Verification Plan**: Testing strategy.
   - **Automated Tests**: Unit tests, etc.
   - **Red Teaming (Failure Cases)**: Boundary values, failure scenarios, and error handling verification.

---

## 3. Escalation Protocol

- **3 Consecutive Failures for the Same AC**: If an AC fails 3 times after attempted fixes, terminate the turn immediately and report the current **[IMPASSE]** reason.
- **Resource Limit**:
  - Evidence recordings (WebP/WebM) must be kept under **10MB**.
  - If a recording exceeds this limit, split the capture or switch to static screenshots.
